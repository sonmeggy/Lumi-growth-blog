# 공개 발행 워크플로우

루미 성장기는 내부 운영 문서와 공개용 문서를 분리해서 발행합니다.

## 권장 흐름

```text
서버 내부 초안
→ Google Docs 검토본
→ 공개용 redaction
→ GitHub Pages 아카이브
→ Tistory 발행본
```

## GitHub Pages의 역할

- Markdown 원본 공개 아카이브
- 기술 문서형 구조
- 장기 보관과 버전 관리

## Tistory의 역할

- 한국어 독자용 읽기 쉬운 발행본
- 검색 유입
- 경험담과 사례 중심 글

## 공개 전 체크리스트

- API 키, 토큰, OAuth 코드 없음
- 개인 이메일 계정명 없음
- 개인 주소와 연락처 없음
- 회사 내부 정보 없음
- 투자 보유 수량과 매입가 없음
- 실제 채팅방 ID 없음
- 민감한 서버 경로 없음

## 표현 원칙

“AI가 스스로 학습했다”보다 “운영 구조가 누적되어 개인화되었다”라고 설명합니다.
